================================================================================
FinishIt - Project Guide
================================================================================

A. PROJECT OVERVIEW
--------------------------------------------------------------------------------
FinishIt is a comprehensive productivity and task management web application.
Unlike traditional to-do lists, FinishIt integrates task tracking, habit 
building, data analytics, and schedule synchronization into a single dashboard.   
Its core philosophy revolves around the "Eisenhower Matrix" — a framework that 
helps users categorize and prioritize tasks based on Urgency and Importance.

Key Features:
1. Eisenhower Matrix Prioritization (Urgent vs. Important)
2. Task CRUD (Create, Read, Update, Delete) with deadlines
3. Habit Tracking with consecutive-day streaks
4. KPI Dashboard and visual Analytics
5. Intelligent Notification Engine (warnings for overdue tasks, missed habits)
6. AI Productivity Assistant (Task Planning, Habit Analysis)
7. Google Calendar Synchronization

B. ARCHITECTURE
--------------------------------------------------------------------------------
FinishIt uses a modular "Client-Driven" architecture.
There is no dedicated custom backend server (like Node.js or Python). Instead, 
the React frontend communicates directly with Supabase via a secure Client API.

Architecture Layers:
- UI Layer: React Components (Pages, Layouts, Components)
- State Layer: React Context (Providers, Context, Hooks)
- Domain/Service Layer: Business logic (services like tasksApi, habitStreaks, etc.)
- Database Layer: Supabase PostgreSQL
- External Integrations: Google Calendar API (OAuth2)

C. TECHNOLOGY STACK

E. FEATURE BREAKDOWN & LOGIC
--------------------------------------------------------------------------------
1. Task Management & The Eisenhower Matrix
Instead of a flat list, tasks are plotted into four quadrants:
- Quadrant 1 (Do First): Urgent & Important.
- Quadrant 2 (Schedule): Important but Not Urgent.
- Quadrant 3 (Delegate): Urgent but Not Important.
- Quadrant 4 (Eliminate): Neither Urgent nor Important.
Logic: Urgency is mathematically calculated. If a deadline is less than 48 hours 
away, `isUrgent` becomes true automatically. Importance is set manually.

2. Habit Tracking & Streaks
Habits are activities that happen regularly (e.g., daily). 
Logic: The system calculates the "Current Streak" by looking backward from today 
and counting how many consecutive days the habit log has `completed = true`.

3. Notification Engine
The system polls data every 60 seconds and derives alerts locally:
- Checks for deadlines < 4 hours (Warning)
- Checks for overdue tasks (Critical)
- Checks for missed habits (Warning)
These generate a visual blinking red dot on the sidebar bell icon.

4. AI Productivity Assistant
The application includes an embedded AI assistant using the Groq API (LLaMA 3). 
Logic: It dynamically gathers all user data (tasks, habits, streaks, matrix priorities) 
and injects it into a structured system prompt, allowing the AI to generate personalized 
productivity plans and weekly summaries instantly.

F. DATA FLOW EXAMPLE (Creating a Task)
--------------------------------------------------------------------------------
1. User types in "Submit Assignment" and clicks "Create" in TaskManagementPage.jsx.
2. The UI checks validation.js to ensure the title isn't blank and length is < 200.
3. The UI extracts the user's ID from the `useAuth()` hook.
4. The API function `createTask()` in tasksApi.js is called.
5. The Supabase client sends securely formatted JSON to the PostgreSQL database.
6. The UI awaits the response, clears the form, and re-fetches the updated list.

G. HOMEPAGE & ROUTING
--------------------------------------------------------------------------------
The application implements public and protected routing architectures:
- `/` maps directly to the LandingPage for public visitors, featuring a premium 
  marketing view.
- `/login` is a `PublicRoute`, meaning users who are already logged in are seamlessly
  redirected to their dashboard to prevent redundant logins.
- `/dashboard` (and others) are wrapped in `ProtectedRoute`. Unauthenticated users 
  are blocked and redirected to login.

K. EXPECTED Q&A
--------------------------------------------------------------------------------
Q: Why use Vite instead of Create-React-App?
A: Create-React-App is deprecated and slow. Vite uses native ES modules, making 
   the development server start instantly and builds significantly faster.

Q: How is data secured if there's no backend?
A: Supabase enforces security at the database level. Every request requires an 
   authentication token, and Row Level Security (RLS) policies ensure users can 
   only query their own data.

Q: How do you calculate streaks?
A: We fetch the historical logs for a habit, sort them chronologically, and iterate 
   backward from the current date. If the sequence of days is uninterrupted, we 
   increment the current streak counter.

Q: How does the application handle state management without Redux?
A: We use React Context (AuthContext, NotificationContext) and custom hooks. This 
   provides a lightweight way to share global state like authentication and 
   notifications without the overhead and boilerplate of Redux.

Q: What is the significance of the 48-hour threshold for task urgency?
A: This is part of our Eisenhower Matrix implementation. If a task is due in 
   less than 48 hours, it's automatically flagged as "Urgent" to encourage 
   immediate action, helping users focus on upcoming deadlines.

Q: How does the Notification Engine ensure alerts stay up-to-date?
A: It uses a polling-based system. The NotificationProvider runs the engine every 
   60 seconds using `setInterval`, checking for upcoming deadlines (<4 hours) 
   and overdue tasks, then updating the UI state and triggering the visual bell.

Q: Why use Vanilla CSS instead of a framework like Tailwind?
A: Vanilla CSS with modern features (CSS Variables, Flexbox/Grid) allows for 
   absolute control over the "premium/hotel lifestyle" aesthetic. It avoids 
   the "generic" look of many frameworks and keeps the CSS bundle highly 
   optimized.

Q: How does the AI Assistant provide personalized advice?
A: Before calling the Groq API, the service gathers the user's real-time data 
   (tasks, habits, streaks). This context is injected into a structured System 
   Prompt, allowing the AI to "see" the user's current productivity state.

Q: What happens if Google Calendar synchronization fails?
A: We track sync status in the `task_calendar_sync` table. Error handlers in the 
   service layer catch API failures, allowing the UI to notify the user without 
   corrupting the primary task data in Supabase.
